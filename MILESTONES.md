## M4. Тематические SEO-тексты для карточек брокеров

**Статус: ПЛАНИРУЕТСЯ**

Кропотливая работа от страницы к странице. Каждая карточка брокера на каждом рейтинге должна содержать **уникальный текст, релевантный именно этой теме**. Это ядро SEO-стратегии.

**Почему это критично:** Google оценивает тематическую релевантность на уровне параграфа. Если карточка брокера на странице "Best ECN Brokers" содержит тот же текст, что на "Best Brokers for Beginners" — это дублированный контент, каннибализация, потеря позиций.

## M7. Технический SEO — предпусковая оптимизация

**Статус: ПЛАНИРУЕТСЯ**

Без этого Google не проиндексирует 365+ страниц корректно.

- [ ] **sitemap.xml** — динамическая генерация для всех 365+ URL
- [ ] **robots.txt** — разрешить всё, кроме /compare/:pair (бесконечные комбинации)
- [ ] **Canonical URLs** — `<link rel="canonical">` на каждой странице
- [ ] **OG + Twitter meta tags** — для шеринга в соцсетях (title, description, image)
- [ ] **404 страница** — кастомная, с поиском и ссылками на популярные разделы
- [ ] **Trailing slashes** — единообразие URL (без trailing slash)
- [ ] **Скорость загрузки шрифтов** — `font-display: swap`, preload для критичных шрифтов
- [ ] **Lazy loading** — изображения, тяжёлые секции ниже fold

## M8. Core Web Vitals и перформанс

**Статус: ПЛАНИРУЕТСЯ**

Google ранжирует по CWV. Текущий бандл 655KB gzip — нужно оптимизировать.

- [ ] **Code splitting** — dynamic import() для страниц (каждая страница = отдельный chunk)
- [ ] **Ленивая загрузка данных** — 38 брокеров не нужны на главной, грузить по запросу
- [ ] **Оптимизация изображений** — WebP, srcset, правильные размеры
- [ ] **Preconnect** — Google Fonts, flagcdn.com
- [ ] **LCP < 2.5s** — largest contentful paint (hero секция)
- [ ] **CLS = 0** — cumulative layout shift (зарезервировать место под шрифты, изображения)
- [ ] **FID < 100ms** — first input delay (минимизировать JS на main thread)
- [ ] Lighthouse audit: целевой скор 90+ по всем метрикам
